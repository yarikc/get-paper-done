<purpose>
Create or refine the paper-scoped author persona.
</purpose>

<required_reading>
- templates/persona.md
- references/questioning.md
</required_reading>

<process>

Read existing `.paper/PERSONA.md` if present.

Check for reusable profiles in `profiles/*.md`. If the user asks to use one, read it and adapt it into the paper-scoped persona. The local `.paper/PERSONA.md` remains authoritative.

Ask only for missing or weak areas:

- persona name or role
- authority posture
- default stance
- primary voice
- energy level
- vocabulary
- argument style
- uncertainty posture
- tone boundaries
- content expectations
- output preferences
- examples that sound right or wrong

Write or update `.paper/PERSONA.md`.

If `.paper/STATE.md` or `.paper/STATE.json` exists, note that the persona was updated and flag downstream artifacts that may need revision.

</process>
