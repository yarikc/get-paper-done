# Technology Lifecycle Management Example

This is a completed anonymized GPD workspace for an imported internal strategy / decision memo about technology lifecycle management in a regulated-enterprise context.

The example shows a second completed workflow shape beyond the data-products example:

- imported-paper recovery without committing the source draft
- brief, audience, and strategy reconstruction from existing messy material
- research that separates official/public evidence from strategic recommendation
- deep outline, rewritten draft, fact-check, review, revision, and export
- clean `gpd validate --semantic` output

## What To Inspect

- `.paper/RESEARCH.json` shows source registry and claim-to-evidence mapping.
- `.paper/OUTLINE.md` shows how the argument was rebuilt before redrafting.
- `.paper/FACT-CHECK.md` shows the main evidence risk: official sources support governed technology evidence, not the exact proposed operating model.
- `.paper/REVIEW.md` shows the revision loop that made the exported draft ready.
- `.paper/exports/FINAL.md` is the generated handoff artifact.

## Privacy Boundary

The original imported draft is intentionally not included. The committed artifacts use generic roles, generic organization context, and anonymized examples. Do not add source drafts, local paths, real people, real organization identifiers, or sensitive role details to this example.

## Known Limitations

This is a reference workflow artifact, not publication advice. A real external version would need approved internal examples, publication-grade citations, and separate legal / compliance review.
